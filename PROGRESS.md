     public" replacement, just differently-scoped.
   - Net recommendation from the research: no clean fix available. Either keep
     `figure._get_renderer()` as-is (current code, with a `type: ignore` + `noqa: SLF001`
     already marking it as a deliberate private-API use), or add a defensive
     try/`canvas.get_renderer()`-first-then-fallback with a clearer error message if both fail —
     the latter is a small, low-risk improvement but has NOT been implemented per the
     no-action instruction. Candidate for a future small PR, not urgent.

## Post-v1 changes (2026-07-02, all user-requested after reviewing real-art gallery output)

- **Guaranteed perimeter margin**: `render/bboxes.py`'s `ensure_minimum_margin(figure, margin)`
  shrinks each axes' position so at least `margin` of clear figure-fraction space exists on
  every side before exclusions are extracted, called from `hook.py._decorate` with
  `_GUARANTEED_MARGIN = 0.14`. Without this, matplotlib's default layout leaves ~2% bottom
  margin and ~6.5% left margin (tick labels sit tight against the plot) vs. ~10-12% top/right —
  cats only ever landed top/right in practice. **This is a deliberate, opted-in departure from
  pure "decorate around the existing layout" behavior** — catplotlib now actively resizes the
  user's plot area a little. PRD Goal 3 updated to carve out this exception explicitly (commit
  `ca44eb1`) rather than let the PRD silently drift from shipped behavior.
- **Denser tiers**: `core/placement.py`'s `_DENSITY_DIVISORS` halved (sparse/normal/chaotic now
  ~6/12/24 cats on the M1 reference canvas, was ~3/6/12).
- **Bigger cats**: `hook.py`'s `size_range` widened `(0.03, 0.08)` → `(0.06, 0.16)` (roughly
  doubled, matching the density-tier bump's proportions). `_GUARANTEED_MARGIN` bumped
  `0.10` → `0.14` in the same change to keep the largest rotated cat (~0.226 footprint at
  size=0.16) reliably fitting without shrink-on-failure dominating.
- Reference numbers ("3/6/12 cats", "size 0.03-0.08") baked into code comments and
  `specs/001-core-placement-engine/{quickstart,data-model}.md` are now stale by ~2x — not swept
  for a full doc update since they're illustrative examples, not enforced contracts. Low
  priority; fix opportunistically if touching those files for other reasons.
- None of these three changes touched `core/placement.py`'s actual exclusion/collision math
  (margin-guarantee lives entirely in `render/`, upstream of `core/`) except the two tuning
  constants — M1's contracts and property tests are otherwise unaffected.

## Known limitations (not blockers, but real gaps)

- **Dependency floors are asserted, not verified.** `pyproject.toml` declares
  `matplotlib>=3.7`, `numpy>=1.24` — set during M0 scaffolding, before most real API usage was
  written. No CI test matrix exists across matplotlib versions. Plausible the true minimum is
  higher for some API actually used.
- **Relies on a private matplotlib API.** `render/bboxes.py` calls `figure._get_renderer()`
  (leading underscore = not part of matplotlib's public compatibility contract). Combined with
  no upper version bound on the matplotlib dependency, a future matplotlib release could break
  this silently. Researched — no public alternative exists — see "Next task" above for the full
  findings and candidate defensive fix (not yet implemented).
- `styles.toml`'s `scale` field is clamped to `min(scale, 1.0)` in `render/artist.py` — can only
  shrink a cat's footprint within the bbox `core/placement.py` reserved, never grow it. Chonk's
  manifest `scale = 1.15` has no visible effect. Would need style-specific scale threaded into
  `core/`'s collision math, conflicting with M1's "styles are opaque strings to core" boundary.
- `ensure_minimum_margin` calls `axes.set_position()` once per decoration; figures using
  `constrained_layout=True` or `tight_layout()` may have their own layout engine re-run during
  the real draw call afterward and override this — untested against that case. Default
  `plt.subplots()` figures (no explicit layout engine) are unaffected.

## Design decisions worth knowing before touching `core/` or `render/`

- `Placement.bbox()` (`core/placement.py`) reserves the AABB of the ROTATED square, not the
  unrotated one (dated addendum in `specs/001-core-placement-engine/research.md`). `render/
  artist.py` draws each cat at its true size within that reserved axes and applies an
  `Affine2D` rotation transform in data-space. Any future placement/rendering geometry change
  must keep the rendered footprint inside what `bbox()` reserves.
- Each cat is its own tiny inset `Axes` (tagged `_catplotlib_cat = True`, excluded from future
  exclusion extraction) — not `AnnotationBbox`/`OffsetImage`, whose point-space `zoom` caused
  real sub-pixel overlaps the chart-matrix test caught but simpler tests didn't.
- `Figure.draw` is patched exactly once, permanently, at import time. `enable()`/`disable()`
  only flip `Config.enabled`, checked first in the wrapper — pure pass-through when `False`,
  which is what makes disabled output byte-identical to unpatched matplotlib.
- `pyproject.toml`'s `[tool.mypy] python_version` is `"3.12"`, not the package's
  `requires-python` floor of `"3.10"` — numpy's bundled stubs need PEP 695 syntax mypy can only
  parse at 3.12+. Not a statement that the package needs 3.12 at runtime.
